# Nightflight OJ Design Notes

这份文档记录《良乡夜航》的公开设计思路，帮助后来者理解玩法模块为什么这样拆、哪些地方可以继续扩展。

## 核心体验

游戏把 Online Judge 的压力感翻译成弹幕生存：

- 样例通过不等于 AC，所以普通敌机和 Boss 会持续抬高弹幕密度。
- 玩家不只是躲弹，还要主动贴弹、冲刺、清屏，才能打出高分。
- 分数不只服务结算，也驱动段位、升级选择和排行榜竞争。

目标手感是“能看懂危险，也能靠技术反打”。Perfect Dash 和判题模式就是这个反打空间。

## 玩家身份

项目只保留一个用户名标识，不再拆分账号和 ID。用户名同时用于：

- 本地 `localStorage` 档案读取。
- 云端排行榜提交。
- 房间同步中的玩家显示名。

这样试玩时没有注册成本，公开部署时也不会引入密码、邮箱或学号之类的隐私字段。

段位由累计总积分映射：

```text
Unrated -> Bronze -> Silver -> Gold -> Platinum -> Diamond -> Red
```

它不影响单局强度，只给玩家一个长期目标。

## 战斗循环

1. 常规敌机生成弹幕和小范围压力。
2. 玩家通过移动、擦弹、冲刺和道具维持生存。
3. 分数到达里程碑或击败 Boss 后进入升级选择。
4. Boss 周期性登场，用更高密度和阶段变化制造高潮。
5. 死亡后提交本局成绩并更新本地/云端档案。

## Boss 设计

Boss 被包装成三类 OJ 压力测试：

| Boss | 主题 | 弹幕重点 |
| --- | --- | --- |
| 隐藏样例：边界风暴 | 边界条件 | 环形扩散和空间压缩 |
| 系统测试：内存漩涡 | 复杂度压力 | 扇形瞄准和节奏切换 |
| 终测机：红名守门人 | 最终验题 | 螺旋弹和高速点名弹 |

Boss 血量下降后进入更高 Phase。Phase 不只是提高数值，也改变玩家需要处理的信息形态。

## 升级池

升级池按角色分成几组：

- **火力：** 提高射速、弹道数量和爆发输出。
- **机动：** 提高移动速度、冲刺等级和 Perfect Dash 收益。
- **协议：** 延长判题模式，强化护盾和 Boss 压制。
- **生存：** 增加生命、护盾和容错。
- **技巧：** 放大擦弹和冲刺收益。

目前升级从 12 张卡里随机给 3 张。这个数量足够形成路线差异，但仍便于玩家在短时间内读懂。

## 输入设计

桌面端以键盘为主，移动端做了两层输入：

- 触摸画布时采用相对拖动，手指不需要压在自机身上。
- 屏幕左侧保留虚拟摇杆，右侧放冲刺、判题和暂停按钮。

手柄走浏览器 Gamepad API，支持摇杆、D-pad、面键、肩键和菜单键。升级、剧情、菜单状态也都能用手柄推进。

## 线上能力

Cloudflare 部分保持轻量：

- D1 存单人榜、双人榜和用户名档案摘要。
- Pages Functions 提供排行榜读写接口。
- Durable Object 负责房间 WebSocket 和玩家状态广播。

当前房间同步偏“轻联机”：广播玩家位置、分数和在线人数。完整双人 Boss 战还需要确定性随机种子、统一暂停/升级时停、Boss 血量仲裁和双人结算签名。

## 可继续扩展

- 完整双人模式：房间准备、同步开局、共享视角、统一时停、双人专属榜。
- 更多 Boss：图论迷宫、DP 状态爆炸、浮点误差审判。
- 更多构筑：近战清弹、护盾反击、能量循环、低血高伤。
- 回放和反作弊：记录输入帧，服务端校验分数区间。
- 移动端细节：横竖屏 HUD、震动反馈、触控灵敏度设置。
